# Cache Behavior Analysis Research

## 概述

本专题研究Apple M2 GPU的缓存层级行为，包括L1、L2缓存大小、缓存行大小以及对性能的影响。

## Apple M2 缓存层级

| 缓存级别 | 大小 | 延迟 | 作用域 |
|---------|------|------|--------|
| L1 | 32 KB | ~1 cycle | Per cluster |
| L2 | ~4 MB | ~10 cycles | Shared |

## 关键发现

### 缓存行大小效应

| 访问模式 | 性能 | 原因 |
|----------|------|------|
| Sequential (1) | ~0.75 GB/s | 充分利用空间局部性 |
| Stride-2 | ~0.33 GB/s | 浪费50%缓存行 |
| Stride-8 | ~0.14 GB/s | 浪费87.5%缓存行 |
| Random | ~0.05 GB/s | 无局部性，每次miss |

### 工作集大小效应

| 工作集大小 | 缓存级别 | 性能 |
|-----------|----------|------|
| 32 KB | L1 | 最优 |
| 256 KB | L1+L2 | ~10%下降 |
| 1 MB | L2 | ~30%下降 |
| 8 MB | DRAM | ~50%下降 |

### 带宽 vs 缓存大小

| Buffer Size | Read BW | 缓存状态 |
|-------------|----------|----------|
| 64KB | 0.05 GB/s | L1 |
| 256KB | 0.17 GB/s | L2 |
| 1MB | 0.50 GB/s | 过渡 |
| 8MB | 1.65 GB/s | DRAM |
| 64MB | 2.11 GB/s | 饱和 |

## 关键洞察

1. **顺序访问最优** - 利用空间局部性，减少缓存miss
2. **跨步访问低效** - stride越大，浪费缓存行越多
3. **随机访问最差** - 无任何局部性
4. **工作集决定缓存级别** - 超过L2会导致显著下降
5. **缓存行64字节** - 访问模式需要考虑对齐

## 优化策略

1. **数据重排** - 确保顺序访问模式
2. **分块** - 将大工作集分成L1/L2大小
3. **预取** - 提前加载数据到缓存
4. **避免跨步** - 使用共享内存中转
